# Thou (iOS) 进度追踪

## 当前阶段

当前处于阶段 1：MVP 收口。

这里的 MVP 定义为“上架前的准备都已就绪”，不是“已经上架”。

## 阶段定义

### 阶段 1：MVP 收口

Thou 侧完成定义：

1. OpenClaw companion 主链稳定可用。
2. iPhone 端能连接、选 agent、选 session、读 history、发消息。
3. Alice 不再进入首发 UI。
4. 真机部署、签名过期后的恢复路径、分发前检查项已明确。
5. GitHub 与 App Store 两条上架准备各自完成。

### 阶段 2：上架

Thou 的上架面有两个：

1. GitHub 公开仓库。
2. App Store。

### 阶段 3：上架后第一阶段

继续收口 OpenClaw companion 的连接体验、状态表达与最小 UI。

### 阶段 4：后续阶段

Alice、工作流层以及更后续的产品关系，先按当前既有判断推进，但不纳入首发完成条件。

## 当前已完成

1. Tailnet 远程 bridge 主链已跑通，iPhone 可完成鉴权、发送、流式回复与历史同步。
2. OpenClaw 页已接入真实 bridge 数据，可读取 agent 列表、按 agent 读取 session 列表、按 `agentId + sessionKey` 读取 history 并发送消息。
3. iOS 端已补齐失活刷新与前台受控自动重连，减少假在线和无限等待。
4. 当前产品判断已收敛为“Tailnet 远程优先，LAN 仅保留底层兼容能力”。
5. OpenClaw 页的首发边界已收敛为：连接、agent 选择、session 选择、history 读取与发送消息。
6. 首发 UI 已收口为 OpenClaw companion，可从 agent 列表进入对话，并从对话页右滑返回 agent 列表；Alice 已从当前可见首发 UI 中隐藏。
7. `Thou (iOS)/` 核心文档目录已从 Thou target 的同步集合中剥离，避免三件套继续被打进 App 包。
8. 当前 iOS 侧仍依赖 ATS 放宽来允许 `ws://...` 明文连接；在仍以开发态 WebSocket 明文链路联调时，这个开关暂时不能直接撤回。
9. Alice 线已有可继续复用的实现结论：
   - OpenRouter 请求链已跑通。
   - 发送前 retrieve、回复后 archive、前台截断的编排主链已入代码。
   - 当前更值得保留的方向是 Methodology A / B、Canonical Impressions 与结构化 ImpressionRecord，而不是继续展开问卷分枝。
10. `Thou/Thou/ViewModels/ChatViewModel.swift` 与 `Thou/Thou/Views/ChatView.swift` 已进一步收成单模 OpenClaw 壳层：不再组合 Alice module，也不再保留 Alice 分支渲染。
11. `Thou/` 仓库根已补齐首发开源骨架：`README.md`、`.gitignore`、`LICENSE`、`CHANGELOG.md`。
12. 本轮已再次完成一轮真机部署：`Thou.app` 已构建到 `.build-device/Build/Products/Debug-iphoneos/Thou.app`，并成功安装到 `李正一的iPhone` 后被 `devicectl` 拉起。

## 当前未完成

1. 真机首发路径还缺最后一轮围绕 `connect -> agent -> session -> history -> send` 的稳定性复测。
2. App Store 与 GitHub 首发准备仍未完全完成。
3. Alice 虽已退出公开壳层，但底层代码资产与后续更彻底的拆分还未处理。
4. `ws://...` 到 `wss://...` 或更窄 ATS 例外的迁移方案还未落实，因此当前还不能安全撤掉 `NSAllowsArbitraryLoads`。

## 当前主线

1. 在已收口的 OpenClaw 首发 UI 上继续做一轮真实手测与小问题归档。
2. 维持文档目录不进入产物资源面的约束，避免后续专题文档再次混入包内。
3. 在首发边界上补齐上架准备，再进入上架阶段。

## 下一步入口

1. 在新包上继续手测 `connect -> agent -> session -> history -> send` 主链，确认 MVP 级稳定性。
2. 把“文档不进 target membership / 不进应用资源面”作为后续整理专题文档时的硬约束执行。
3. 在继续依赖 `ws://...` 的阶段保留 ATS 放宽；只有当连接链路改成 `wss://...`，或已收敛为足够窄的 ATS 域名例外后，才允许撤掉当前的 `NSAllowsArbitraryLoads`。
4. 后续若要降低对 Tailnet / Tailscale 的依赖，应把 Paseo 作为重点参考对象，优先研究它在远程入口、网络穿透或设备接入上的公开代码与产品取舍，再决定 Thou 是否要跟进这条方向。
5. 继续推进 TestFlight / GitHub 首发准备；App Store 正式提审延后到网络与资料进一步收口后再判断。
