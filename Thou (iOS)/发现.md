# 发现 (Thou iOS)

## 当前有效事实

- 当前工作区是 multi-root workspace；`Thou/` 与 `OpenClaw插件/` 是真实项目，`Reference/` 是共享参考库。
- OpenClaw companion 当前已接入真实 bridge 数据，不再只是本地占位 UI。
- OpenClaw 页当前产品边界仍是 `agent inbox -> 单 agent 对话页 -> session 切换`；手机端第一版不承担 agent 创建与 session 创建。
- Tailnet 远程已再次验证可用；当前主连接心智应继续保持远程优先，不再把 `.local` 或 LAN IPv4 当成主路径。
- iOS 端已补齐心跳失败、ping/send 失败后的主动断线处理，用于避免“假在线 + 一直等待”。
- 前台恢复时的受控自动重连已补上；MVP 阶段不追求后台持续保活。
- 仅重新构建 `native-plugin` 不足以让 Thou 看到新 bridge 逻辑；宿主若由 launchd 托管，必须执行 `openclaw gateway restart`。
- 当前首发边界明确要求 Alice 不进入 UI，但现有代码根壳仍保留 Alice / OpenClaw 双模式切换，因此这一点尚未完成。
- 手机上当前安装包再次不可用时，优先怀疑 7 天开发签名或设备信任失效，而不是先怀疑业务代码主链。
- 当前已验证过一种典型失败形态：真机构建成功、安装成功，但 `xcrun devicectl device process launch` 仍会因 `invalid code signature, inadequate entitlements or its profile has not been explicitly trusted by the user` 被系统拒绝；这说明设备端信任需要单独确认。
- 当前已再次确认：若用户在 iPhone 端完成开发者信任后，先前“已安装但无法启动”的 Thou 可恢复正常使用；因此这类问题的最短排查顺序应是“真机构建 -> 安装 -> 启动 -> 若仅启动失败则优先检查设备端信任”，而不是继续重复重装。
- 对 iOS 项目而言，真机部署完成的最低确认信号仍是三件事：构建成功、安装成功、App 被实际拉起。
- 当前已发现并处理：`Thou (iOS)/` 核心文档目录曾被 Xcode 的 target 同步集合带进应用资源面；真机构建日志里出现过把 `发现.md` 复制进 `.app` 包、并清理旧 Alice 草稿文件残留的记录。现已将该目录从 Thou target 的同步集合中剥离，后续无论是核心文档还是专题文档，都不应继续通过 target membership 混进 App 资源面。
- 当前已进一步确认首发壳层收口：`Thou/Thou/ViewModels/ChatViewModel.swift` 已不再组合 Alice module，`Thou/Thou/Views/ChatView.swift` 也已删除 Alice 分支渲染；当前公开聊天壳层只承接 OpenClaw companion。
- 当前已进一步确认仓库首发骨架：`Thou/` 根目录已补齐 `README.md`、`.gitignore`、`LICENSE`、`CHANGELOG.md`，可作为独立 GitHub 仓库的最小公开面。
- 当前已再次确认真机部署最短闭环：`xcodebuild -project Thou.xcodeproj -scheme Thou -configuration Debug -destination 'id=00008150-001874643E02401C' -derivedDataPath .build-device -allowProvisioningUpdates build` 产出 `.build-device/Build/Products/Debug-iphoneos/Thou.app` 后，可用 `xcrun devicectl device install app ...` 安装，并用 `xcrun devicectl device process launch --device 00008150-001874643E02401C Zheng.Thou` 成功拉起。
- 当前已进一步确认一套更合理的 Thou 真机部署顺序：先用 `xcrun xcdevice list` 或等效命令确认真机 identifier；再直接走 `xcodebuild -> devicectl install -> devicectl launch` 这条闭环。若构建命令的终端输出丢失，不要立刻重跑整条链，而应先分别核对三件事：`.build-device/Build/Products/Debug-iphoneos/Thou.app` 是否已生成、设备是否已安装 `Zheng.Thou`、`devicectl` 是否能单独拉起该 bundle。这样能更快区分是构建失败、安装失败，还是启动/信任失败。
- Alice 线当前仍保留一批可继续复用的技术结论：
  1. OpenRouter 主链已跑通。
  2. retrieve 与 archive 已入编排链。
  3. 更该保留的方向是 Methodology A / B、Canonical Impressions、结构化 ImpressionRecord，而不是继续扩问卷旁支。
- 首发阶段不要求先证明 Alice 的完整产品命题，也不要求先证明“虚拟团队工作法”。
